Lesson 1: The Basics of the Web and HTML


The Web As We Know It

The way we view the web was due to Tim Berners-Lee developing the World Wide Web (aka The Web) in 1989 and HTML (Hyper Text Markup Language) in 1990.

The web contains billions of HTML documents. HTML is the skeleton of every web page, and web pages can be connected with hyperlinks (aka links).

There are four major pieces in order to request and retrieve a website, which are a Browser, the Internet, HTTP, and Servers. The browser you use makes a request to the internet asking for website you entered in your browser, the internet communicates that request using HTTP to the server that is hosting the HTML file of the website you requested. The server then executes that request by communicating to the internet using HTTP, and the internet displays the website on your browser. Please see the following image for visual understanding:


Understanding HTML

As stated previously HTML is the skeleton of every web page, and in essence it is the foundation of the web.

HTML has a number of tags that be can be used to alter the look of text content or reference other documents such as: images or videos.

Some of the common tags are 'p' for paragraph, 'b' for bold, and 'em' for emphasis. All of those tags require an opening and closing tag utilizing angle brackets, it is important to remember to use the slash '/' in the closing tag.

Another useful tag is the 'a' for anchor, the anchor tag allows a programmer to place links to other web pages. The anchor tag requires a programmer to use an attribute that's referred to href. An additonal tag that requires an attribute is the image 'img' tag, it requires the src attribute. The image tag allows a programmer to add images to a webpage. A programmer should remember when using this tag to also include the alt attribute to enter text of the image, it will be helpful to users if the link is broken. A programmer should recognize that the 'img' tag doesn't require a closing tag, and neither does the breaking 'br' tag. Both of those tags are void tags, since they don't have any content.

Every HTML element is either inline or block. The diffence between the two is that a block has an invisible box surrounding the contexts and a inline doesn't.


Key Pieces of Advice When Programming

It is important to understand that computers will execute exactly what a programmer codes, they will not assume what a programmer may have meant to code.

Don't attempt to remember all of the elements, being forgetful is forgivable just know what resources to use to find help such as Google or Stack Overflow

Finally, in order to become a better programmer, it is necessary to constantly practice.



Lession 2: Creating a Structured Document with HTML


An Analogy of a Web Page

HTML is the foundation and structure of a home.

CSS is the style of a home. Such as the color of walls or whether it has brick or siding.

DOM (document object model) is the blueprint of a home, it allows someone to view the structure of a web page (doesn't provide the actual code). This can be viewed by using the Developer Tools in a browser, there are two panels an: Element Panel and Styles Panel. In the Elements Panel sideway triangles may appear representing an element. If you click on the triangles, they will reveal further elements, that relate to the previous element, this will provide the appearance of a tree-like structure. The DOM is built by the browser that's being used. The Styles Panel provides information about specific details of an element such as: pixel size, color, font type, border height and width, etc.


I See Boxes

Web pages are structured by a series of boxes, the div tag is best to create these boxes. This can be seen using the Developer Tools in a browser. When creating a div tag give it a place holder in order to remember which box it refers to. It's important to give each div tag a class attribute in order to give specific style flexibility for each box when using CSS.

Having boxes as the layout of web pages makes it easier for developers to rearrange a page and give users a new viewing experience.

It is important to get in the habit that when given a new project to begin drawing boxes of the layout of any web page, this is known as boxify the design. It not important to include the content until later. An example of a boxified design with place holders can be seen in the following image.

Lession 3: Adding CSS Style to HTML Structure


The Power of CSS

CSS can be used three ways: First, it can be included between the head element of a HTML file, this should only be used for small projects. Second, it can be included by creating a separate CSS file and linking it to the related HTML file, this is the most preferred. Lastly, it can be included inline within the HTML file. This is never suggested, because it can cause a programmer valuable time if they need to make changes to the style of the page. Which would defeat the purpose of using CSS.

The most meaningful use of CSS for a programmer is to not have to repeat themself when applying the same style to the same elements throughout their HTML document. It also allows a programmer to make style changes quickly, by just updating their attribute and/or value.
